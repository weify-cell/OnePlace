# PM 需求分析 handoff

## 模块信息
- 版本号：v1.9
- 模块名：notes-preview-mode
- 启动日期：2026-04-01

## 需求结论

笔记详情页增加预览/编辑双模式：打开时默认纯预览，点击"编辑"或标题进入分屏编辑模式，点击"取消编辑"返回预览（未保存内容弹窗确认）。笔记列表支持卡片/列表切换视图。

## 关键决策

| # | 问题 | 选择 | 说明 |
|---|------|------|------|
| 1 | 取消编辑后如何返回预览 | A | 显示「取消编辑」按钮，点击恢复纯预览 |
| 2 | 预览模式下按钮数量 | B | 双按钮：编辑 + 返回列表 |
| 3 | 如何切换回预览 | A | 「完成编辑」按钮，点击保存并返回预览 |
| 4 | 取消时未保存内容处理 | B | 弹窗确认「有未保存的更改，确定要放弃吗？」 |
| 5 | 点击标题行为 | A | 点击标题直接进入编辑模式 |
| 6 | 编辑器类型 | A | 沿用 CodeMirror Markdown 编辑器（v1.8） |
| 7 | 笔记列表视图 | C | 卡片/列表双模式，用户可切换，记住偏好 |
| 8 | 默认视图 | A | 默认卡片模式 |

## 功能范围

### 纳入
- 笔记详情页预览模式（默认）
- 编辑按钮进入分屏编辑模式
- 点击标题进入编辑模式
- 「完成编辑」按钮保存并返回预览
- 「取消编辑」按钮 + 弹窗确认
- 笔记列表卡片/列表切换 + 偏好持久化

### 排除
- 编辑器本身改动（沿用 v1.8 实现）

## 交互流程

### 预览模式
- 页面显示纯预览（无编辑器）
- 顶部工具栏：「编辑」按钮 + 「返回列表」按钮
- 点击标题 → 进入编辑模式

### 编辑模式
- 左侧 CodeMirror 编辑器 + 右侧实时预览
- 顶部工具栏：「完成编辑」按钮 + 「取消编辑」按钮 + 「返回列表」按钮
- 点击「完成编辑」→ 保存内容 → 返回预览模式
- 点击「取消编辑」→ 若有未保存更改 → 弹窗确认 → 返回预览模式
