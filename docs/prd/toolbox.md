- [x] **JSON 格式化工具**

#### 2. 图片工具 Tab
- [x] **图片 Base64 互转工具**

  - Base64 → 图片：输入 Base64 字符串，预览并下载图片
  - 输入方式：拖拽上传、点击选择文件、粘贴剪贴板图片
  - 输出格式选项：
    - Data URI 格式（`data:image/png;base64,...`）
    - 纯 Base64 字符串
  - 图片预览：转换后实时预览图片
  - 复制结果：一键复制 Base64 结果
  - 下载图片：从 Base64 下载原图
  - 文件信息：显示文件名、大小、尺寸、格式

## 技术实现建议

### 纯前端实现
- 无需后端接口，所有逻辑在浏览器完成
- JSON 处理使用原生 `JSON.parse`/`JSON.stringify`
- 图片处理使用 `FileReader` API 读取文件
- 剪贴板操作使用 `navigator.clipboard` API

### 代码折叠方案（JSON 工具）
**方案 A（推荐）**：使用 Monaco Editor（VS Code 同款编辑器）
- 优点：功能强大，自带折叠、语法高亮、错误提示
- 缺点：包体积较大（约 1MB+），需要按需加载

**方案 B**：使用 CodeMirror 6
- 优点：轻量，插件丰富
- 缺点：配置相对复杂

**方案 C**：简单实现（如无编辑器库）
- 仅使用 textarea + 行号显示
- 折叠功能用缩进视觉提示代替
- 或本期暂不提供折叠功能

### 图片处理
- 文件读取：`FileReader.readAsDataURL()`
- 图片信息：创建 `Image` 对象获取尺寸
- 下载功能：创建 `a` 标签触发下载

### 状态管理
- 无需 Pinia Store（无持久化需求）
- 每个工具页面使用本地 state 管理
- 刷新页面即清空所有数据

### 技术实现建议

#### diff 算法选择
- **方案 A（推荐）**：使用 `diff` 库（npm）的字符级比较
  - 优点：成熟稳定，字符级支持，开源免费
  - 缺点：需引入依赖（约 50KB）
  - 库：`diff` (https://www.npmjs.com/package/diff)

- **方案 B**：使用 `jsdiff` 库
  - 优点：功能全面，支持多种 diff 模式
  - 缺点：包体积较大

- **方案 C**：手写简易 LCS 算法
  - 优点：无依赖
  - 缺点：实现复杂，边缘情况多

#### 差异渲染
- 使用 Vue 响应式渲染，将 diff 结果转为渲染指令
- 每个字符包装为 `<span>`，根据类型添加不同 class
- 差异导航使用 `scrollIntoView` 定位

#### 性能考虑
- 超过 10000 字符的文本比较时显示加载指示器
- 使用 `requestAnimationFrame` 分批渲染差异结果
