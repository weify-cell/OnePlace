# v1.2 笔记文件夹 + 标签系统 PRD

## 背景与目标

现有笔记模块缺乏组织能力，用户无法对笔记分类或打标签，搜索效率低。
本期目标：为笔记引入**单层文件夹**和**独立标签**两套正交的组织机制，提升笔记的检索和管理体验。

---

## 功能清单（优先级 P0/P1）

### 文件夹功能

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 文件夹 CRUD | 创建、重命名、删除文件夹；单层平铺，不支持嵌套 |
| P0 | 笔记归属文件夹 | 笔记可属于一个文件夹，或不属于任何文件夹（视为"全部"） |
| P0 | 侧边栏展示文件夹列表 | 笔记侧边栏新增文件夹区域，点击文件夹筛选该文件夹下的笔记 |
| P0 | 侧边栏"全部笔记"入口 | 默认选中，展示所有未删除笔记 |
| P1 | 创建笔记时选择文件夹 | 新建笔记弹窗/面板支持选择目标文件夹 |
| P1 | 笔记编辑页移动文件夹 | 在笔记详情页可修改所属文件夹 |
| P1 | 删除文件夹时处理笔记 | 删除文件夹后，其下笔记的 folder_id 置为 NULL（笔记不删除） |

### 标签功能

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 笔记编辑时添加/删除标签 | 在笔记详情页支持输入标签、删除已有标签 |
| P0 | 标签数据结构 | 以 JSON 数组存储在 notes 表的 TEXT 列，与 Todo 标签实现一致但数据完全独立 |
| P0 | 按标签筛选笔记 | 列表页（或侧边栏）支持点击标签筛选，使用 SQLite json_each() 查询 |
| P1 | 标签聚合展示 | 侧边栏或列表页顶部展示当前所有已使用标签，点击快速筛选 |

## 验收标准（AC）

### 文件夹

**AC-F1 创建文件夹**
```
Given 用户在侧边栏点击"新建文件夹"
When 输入文件夹名称并确认
Then 文件夹出现在侧边栏列表；数据库 folders 表新增一条记录
```

**AC-F2 重命名文件夹**
```
Given 侧边栏存在文件夹"工作"
When 用户右键或点击编辑图标，输入新名称"项目"并确认
Then 侧边栏显示"项目"；数据库对应记录 name 字段更新
```

**AC-F3 删除文件夹**
```
Given 文件夹"工作"下有 3 篇笔记
When 用户删除文件夹"工作"
Then 侧边栏移除该文件夹；3 篇笔记的 folder_id 置为 NULL，笔记本身不删除
```

**AC-F4 按文件夹筛选笔记**
```
Given 侧边栏存在文件夹"工作"
When 用户点击"工作"
Then 笔记列表仅显示属于"工作"文件夹的笔记
```

**AC-F5 笔记移动文件夹**
```
Given 笔记当前属于文件夹"工作"
When 用户在笔记详情页将文件夹改为"生活"
Then 笔记从"工作"列表消失，出现在"生活"列表
```

### 标签

**AC-T1 添加标签**
```
Given 笔记详情页处于编辑状态
When 用户在标签输入框输入"会议"并按 Enter
Then 标签"会议"显示在笔记标签区域；数据库 notes.tags 列更新为包含"会议"的 JSON 数组
```

**AC-T2 删除标签**
```
Given 笔记已有标签["会议", "重要"]
When 用户点击"会议"标签的删除按钮
Then 标签区域仅剩"重要"；数据库更新对应 JSON 数组
```

**AC-T3 按标签筛选**
```
Given 存在多篇笔记，其中 2 篇含标签"重要"
When 用户点击标签"重要"进行筛选
Then 笔记列表仅显示含"重要"标签的笔记
```

**AC-T4 标签数据隔离**
```
Given Todo 模块存在标签"工作"
When 用户在笔记标签区域搜索
Then 不应出现来自 Todo 的标签建议；两者数据完全独立
```

## 不在范围内（Out of Scope）

- 文件夹多级嵌套（本期仅单层）
- 文件夹排序/拖拽
- 标签颜色自定义
- 笔记与 Todo 共享标签池
- 文件夹/标签的批量操作
- 文件夹共享或权限控制
- 标签重命名（删除后重新添加）
