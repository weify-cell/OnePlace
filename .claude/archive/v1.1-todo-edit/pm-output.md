- 迭代：v1.1
- 模块：todo-edit
- 日期：2026-03-21

## 功能清单

- [P0] 功能1：TodoEditModal 弹窗组件 — 复用 TodoCreateModal 字段（标题、描述、优先级、类型、截止日期、标签），新增 status 字段，通过 props 接收初始数据，提交调用 `todoStore.updateTodo()`
- [P0] 功能2：列表行编辑入口 — 鼠标悬停在待办行时显示"编辑"图标按钮，点击打开 TodoEditModal
- [P0] 功能3：编辑后列表刷新 — `updateTodo()` 已实现乐观更新（原地替换 items），编辑弹窗关闭后无需额外 fetchTodos
- [P1] 功能4：快捷状态切换 — 列表行复选框直接调用已有 `toggleStatus()`，不走编辑弹窗
- [P1] 功能5：标题行内编辑 — 双击列表行标题进入行内 input，失焦时保存，避免打开弹窗的步骤
- [P2] 功能6：键盘快捷键 — 选中行后按 E 键打开编辑弹窗；按 Esc 关闭

## 验收标准

**AC-1 打开编辑弹窗**
Given 用户在待办列表页，待办行存在
When 鼠标悬停到该行，点击编辑图标
Then 弹窗打开，字段预填该条待办的当前值（标题、描述、优先级、类型、截止日期、标签、状态）

**AC-2 保存编辑**
Given 编辑弹窗已打开并修改了至少一个字段
When 点击"保存"按钮
Then 调用 PUT /api/todos/:id，列表中对应行数据即时更新，弹窗关闭，显示"保存成功"提示

**AC-3 标题不能为空**
Given 编辑弹窗已打开
When 清空标题后点击"保存"
Then 显示"请输入标题"错误提示，请求不发出，弹窗保持打开

**AC-4 取消编辑**
Given 编辑弹窗已打开并修改了字段
When 点击"取消"或按 Esc
Then 弹窗关闭，列表数据不变

**AC-5 快捷状态切换**
Given 待办状态为 todo
When 点击行复选框
Then 调用 toggleStatus()，状态切换为 done，复选框即时选中；再次点击恢复 todo

## 边界情况和已知风险

| 风险 | 说明 | 建议 |
|------|------|------|
| 并发编辑 | 两个标签页同时编辑同一条待办 | 后端以最后一次写入为准，前端暂不处理冲突提示（P2） |
| 标签缓存失效 | 编辑后标签变更，allTags 未刷新 | 编辑成功后调用 fetchAllTags() |
| 截止日期时区 | 后端存储 yyyy-MM-dd，DatePicker value-format 已对齐 | 保持一致，无风险 |
| updateTodo 乐观更新失败 | 若接口报错，items 已被替换；需 catch 后重新 fetchTodos | 后端保证 400/500 正确返回 |
| TodoCreateModal 复用 | 现有组件无 modelValue prop，改造会影响新建流程 | 推荐新建独立 TodoEditModal 组件 |
